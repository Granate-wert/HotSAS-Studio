# HotSAS Studio Roadmap

## Финальная идея программы

**HotSAS Studio — Hardware-Oriented Schematic Analysis & Simulation Studio** — это инженерная desktop-среда для схемотехники.

Итоговая программа должна объединять:

```text
схемный редактор
+ инженерный калькулятор
+ библиотеку формул
+ схемные заготовки
+ E-ряды
+ реальные компоненты
+ symbol + footprint
+ SPICE/ngspice симуляцию
+ графики
+ анализ выделенного участка цепи
+ отчёты
+ экспорт в EDA workflow
```

Текущий README уже фиксирует базовую идею: приложение для schematic analysis, formula-driven circuit templates, SPICE-oriented workflows и report generation; текущий v1 vertical slice сфокусирован на RC low-pass demo project: создание проекта, отрисовка схемы из backend DTO, расчёт `fc`, подбор E24, SPICE netlist, mock AC simulation, график, Markdown/HTML report и сохранение JSON. ([GitHub][2])

# 8. Roadmap v6: optional PCB-aware expansion

Ты говорил, что PCB пока не планируется. Поэтому этот этап **опциональный и далёкий**.

## v6.0 — PCB Data Awareness, not PCB Editor

Минимум:

```text
- component footprint assignment;
- pin-pad validation;
- package dimensions;
- assembly outline;
- courtyard metadata;
- export hints.
```

Не делать:

```text
- трассировку;
- слои платы;
- DRC платы;
- vias;
- Gerber.
```

---

## v6.5 — Optional PCB Module

Только если проект сильно вырастет.

Функции:

```text
- PCB layout model;
- board outline;
- footprints placement;
- nets;
- simple DRC;
- KiCad PCB export.
```

## v7.0 — Manufacturing Export

Только после PCB module:

```text
- Gerber;
- Excellon drill;
- pick-and-place;
- assembly BOM;
- IPC-2581 later.
```

До этого Gerber делать нельзя, потому что Gerber — это производственный формат платы, а не схемы.

# 10. Что делать прямо сейчас

Текущее следующее действие:

```text
Не идти сразу в v1.2.
Сначала закрыть v1.1.5.
```

Потому что сейчас есть незавершённый переход:

```text
FormulaRegistry/FormulaLibrary UI уже частично есть,
но generic FormulaEngine/API/formatting/YAML/E-series надо довести до стабильного состояния.
```

Правильный ближайший порядок:

```text
1. v1.1.5 — completion gate + exact E-series.
2. Проверить cargo test / npm build.
3. Только потом v1.2 — .circuit project package.
```

После v1.1.5 проект должен иметь устойчивую базу:

```text
backend architecture чистая
+ formula packs runtime
+ generic formula calculation
+ exact E-series
+ tests
+ UI Formula Library через backend
+ RC vertical slice сохранён
```

Это будет нормальная точка, от которой можно начинать превращать HotSAS Studio из архитектурного прототипа в настоящую инженерную программу.
