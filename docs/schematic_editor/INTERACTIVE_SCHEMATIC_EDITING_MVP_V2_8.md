# HotSAS Studio v2.8 — Interactive Schematic Editing MVP

## Goal

Transition from foundation schematic editing (individual backend commands) to a cohesive user-facing workflow: build a simple circuit on canvas, connect pins, edit parameters, validate, preview netlist, save/load, with undo/redo support.

## What the user can now build

A simple RC circuit manually:

- Voltage Source → Resistor → Capacitor → Ground
- Set R/C values
- Connect pins
- Run validation
- Generate netlist preview
- Save as `.circuit`
- Reopen and see the same schematic

## Architecture rules

- **React Flow remains view adapter only.** All mutations go through the backend facade.
- **Backend (Rust) remains the single source of truth.** No frontend-only persistent schematic state.
- **Undo/redo is snapshot-based, session-local, bounded to 50 actions, and not persisted.**

## Component placement flow

1. User selects a component from the **Placeable Component Palette**.
2. Tool mode becomes `place`.
3. User clicks on the canvas.
4. Frontend sends `place_schematic_component` with `component_definition_id` + position.
5. Backend creates the instance, returns updated `ProjectDto`.
6. Frontend renders the updated schematic from `ProjectDto`.

## Wire tool flow

### Form-driven connection (v2.8 MVP)

- User clicks **Connect** button or uses `ConnectionPanel`.
- Selects source/target components and pins from dropdowns.
- Frontend sends `connect_schematic_pins`.
- Backend validates pin refs, creates wire + net.
- Frontend renders edge from `ProjectDto.wires`.

### Interactive canvas drag-to-connect (v2.8 added)

- `SchematicCanvas` uses React Flow `onConnect`.
- Handles on each pin node have `isConnectableStart` and `isConnectableEnd` enabled.
- User can drag from one pin handle to another.
- `onConnect` callback invokes `onConnectPins` backend command.
- React Flow draws the temporary edge during drag automatically; no custom `WireToolOverlay` is required.

> **Note:** `WireToolOverlay` was planned as a custom visual overlay, but React Flow’s built-in edge-drag behavior covers the wire-drawing visual feedback. The component was intentionally omitted to avoid duplicating React Flow internals.

## Delete wire flow

1. User selects a wire (edge) on canvas or in the selection panel.
2. Clicks **Delete Wire** in `SchematicSelectionInspector`.
3. Frontend sends `delete_schematic_wire`.
4. Backend removes wire, cleans up net connected_pins and component connected_nets.
5. Frontend re-renders without the edge.

## Quick parameter edit flow

1. User selects a component on canvas.
2. `SchematicSelectionInspector` shows editable fields via `QuickParameterEditor`.
3. User types a new value and clicks **Update**.
4. Frontend sends `update_schematic_quick_parameter`.
5. Backend validates and updates the model.
6. Dirty state becomes true.

## Undo/redo

- Snapshot-based in `HotSasApi` facade.
- Bounded stack (50).
- Covers: add component, move, delete, connect, delete wire, rename net, update parameter.
- **Limitation:** session-local only; not persisted in `.circuit`.

## ERC / validation

- `ErcIssuePanel` displays warnings/errors from backend validation.
- Updates after every mutating edit.
- `CircuitValidationPanel` remains for explicit validation runs.

## Netlist preview

- `NetlistPreviewPanel` shows SPICE-like netlist generated by backend.
- Uses existing backend netlist generation service.
- Frontend never generates netlist.

## Save/load roundtrip

- All interactive edits (placed components, positions, wires, nets, parameters) are preserved in `.circuit`.
- Verified by `save_load_roundtrip_preserves_interactive_edits` Rust test.

## Limitations

- Not a full KiCad/Altium editor.
- No PCB editor, autorouter, hierarchical schematic, symbol editor, bus system.
- No production-grade ERC.
- No persistent undo/redo history.
- Wire geometry is optional; if missing, frontend renders straight line between pin positions.
