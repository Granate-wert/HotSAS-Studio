# HotSAS Studio v2.7 — CLI / Headless Mode Foundation Acceptance Matrix

## Objective

Verify that the `hotsas-cli` binary provides a stable, tested, headless interface to all core backend operations without duplicating business logic.

---

## Acceptance Criteria

### AC-1: CLI crate exists and compiles

| Check | Status |
|---|---|
| `engine/cli/` crate exists with `Cargo.toml` | ✅ |
| `cli` listed in `engine/Cargo.toml` workspace members | ✅ |
| `cargo check -p hotsas_cli` passes | ✅ |
| `cargo build -p hotsas_cli --release` produces binary | ✅ |

### AC-2: help and version commands

| Check | Status |
|---|---|
| `hotsas-cli --help` prints usage with all subcommands | ✅ |
| `hotsas-cli --version` prints version string | ✅ |
| Help mentions: validate, formula, netlist, export, simulate, library | ✅ |

### AC-3: validate command

| Check | Status |
|---|---|
| `hotsas-cli validate <valid.circuit>` returns exit code 0 | ✅ |
| `hotsas-cli validate <invalid.circuit>` returns exit code 2 | ✅ |
| `--json` flag returns valid JSON with status field | ✅ |

### AC-4: formula command

| Check | Status |
|---|---|
| `hotsas-cli formula ohms_law V=10 I=0.5 R=1k` returns result | ✅ |
| `hotsas-cli formula rc_low_pass_cutoff R=10k C=100n --json` returns valid JSON | ✅ |
| Missing variables return exit code 2 | ✅ |
| Unknown formula returns exit code 2 | ✅ |

### AC-5: netlist command

| Check | Status |
|---|---|
| `hotsas-cli netlist <project.circuit>` prints netlist to stdout | ✅ |
| `hotsas-cli netlist <project.circuit> --out file.cir` writes to file | ✅ |
| `--json` flag returns valid JSON with netlist content | ✅ |
| Missing project returns exit code 2 | ✅ |

### AC-6: export command

| Check | Status |
|---|---|

| `hotsas-cli export <project.circuit> html` returns HTML content | ✅ |
