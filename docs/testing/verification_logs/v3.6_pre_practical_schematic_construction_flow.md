## Root Cause Note

During full Rust verification, `missing_required_parameter_returns_error` initially failed because validation still matched old definition ids (`resistor`, `capacitor`, `voltage_source`) while the demo schematic uses `generic_resistor`, `generic_capacitor`, and `generic_voltage_source`. The validation service now accepts both id families. The focused failing test and full Rust suite pass after that fix.
