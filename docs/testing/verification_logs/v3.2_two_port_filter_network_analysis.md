## Implemented In This Pass

- Core domain models for two-port / filter network analysis:
  `FilterAnalysisScope`, `CircuitAnalysisPort`, `FrequencySweepSettings`,
  `FilterAnalysisMethod`, `DetectedFilterKind`, `FilterNetworkAnalysisRequest`,
  `FilterNetworkAnalysisResult`, `FilterSweepPoint`, `FilterMetricValue`,
  `FilterAnalysisDiagnostic`.
- `TwoPortFilterAnalysisService` with request validation, port suggestion,
  mock/template/NGSPICE analysis orchestration, filter-kind detection,
  cutoff/peak-gain/impedance metric estimation, CSV export, and
  `ReportSection` generation for advanced reports.
- `AppServices` integration with accessor for `TwoPortFilterAnalysisService`.
- API DTOs mirroring all core models with `From` implementations.
- API facade methods: `suggest_filter_analysis_ports`,
  `validate_filter_network_analysis_request`, `run_filter_network_analysis`,
  `get_last_filter_network_analysis`, `clear_last_filter_network_analysis`,
  `export_filter_network_analysis_csv`,
  `add_filter_network_analysis_to_advanced_report`.
- Tauri commands for all 7 facade methods.
- Frontend TypeScript types, API wrappers, and Zustand store additions
  for filter analysis state.
- CLI `hotsas-cli filter-analyze <project.circuit> [--method <method>] [--out <path>]`.
- Rust API integration tests: 7 tests covering state errors, mock analysis
  with points/metrics, get/clear last result, CSV export without result,
  and advanced report without result.
