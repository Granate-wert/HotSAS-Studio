# 6. Блок C — исправить HTML escaping

Проверить:

```text
engine/adapters/src/lib.rs
MarkdownReportExporter::export_html
```

Нужно, чтобы HTML-экспорт безопасно экранировал:

```text
& -> &amp;
< -> &lt;
> -> &gt;
```

Допустимая реализация:

```rust
fn escape_html(input: &str) -> String {
    input
        .replace('&', "&amp;")
        .replace('<', "&lt;")
        .replace('>', "&gt;")
}
```

Тест:

```text
input:
<script>alert(1)</script>

HTML output НЕ должен содержать:
<script>alert(1)</script>

HTML output должен содержать:
&lt;script&gt;alert(1)&lt;/script&gt;
```

Если текущий тест ожидает активный `<script>` — исправить тест.

Не добавлять полноценный Markdown-to-HTML renderer. На этом этапе достаточно безопасного HTML wrapper / `<pre>`.

# 12. Блок I — HotSasApi facade

Проверить:

```text
engine/api/src/facade.rs
```

Должен быть метод:

```rust
calculate_formula(

) -> Result<FormulaEvaluationResultDto, ApiError>
```

Поведение:

```text
1. Проверить formula_id.
2. Найти FormulaDefinition в FormulaRegistry.
3. Для каждой variable из FormulaDefinition:
   - взять значение из request.variables;
   - если unit в request указан — использовать его;
   - если unit не указан — использовать unit из FormulaDefinition;

5. Вернуть FormulaEvaluationResultDto.
```

```text
- formula not found -> not_found / formula_not_found
- missing variable -> invalid_input
- invalid unit -> invalid_input
- unsupported expression -> formula_error / port_error
```

React должен получить ошибку через Tauri как строку или structured DTO, в зависимости от текущей архитектуры. Не надо сейчас переписывать всю error system.
