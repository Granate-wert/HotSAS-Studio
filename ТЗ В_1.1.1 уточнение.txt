# 2. Что именно нужно сделать

## 2.1. Rust formatting

Нужно привести весь Rust-код в `engine/` к нормальному виду через стандартный `rustfmt`.

Выполни:

```bash id="loznwj"
cd "D:\Документы\vscode\HotSAS Studio\engine"
cargo fmt
cargo fmt --check
cargo test
```

Если `cargo fmt --check` падает, сначала исправь форматирование, потом снова проверь.

## Требования к результату

Файлы Rust не должны быть “одной длинной строкой”. Особенно проверить:

```text id="bk9k75"
engine/core/src/lib.rs
engine/core/src/models.rs
engine/core/src/value.rs
engine/core/src/preferred_values.rs
engine/core/src/templates.rs
engine/core/src/circuit_query.rs

engine/application/src/lib.rs
engine/application/src/errors.rs
engine/application/src/services/*.rs

engine/adapters/src/lib.rs

engine/api/src/lib.rs
engine/api/src/dto.rs
engine/api/src/error.rs
engine/api/src/facade.rs

engine/ports/src/lib.rs
```

Если файл после `cargo fmt` всё равно остаётся одной строкой, значит он был записан без нормальных переносов строк или rustfmt не может его нормально обработать. В таком случае нужно вручную привести его к нормальному многострочному виду, не меняя логику.

## Важное правило

Не менять публичные имена функций, DTO, команд и моделей, если это не требуется для сборки.

## 3.1. Установить Prettier

```bash id="s792z7"

```

Не добавлять ESLint на этом этапе, если он не установлен.

---

## 3.2. Добавить `.prettierrc`

Создать файл:

```text id="lpk0jb"
apps/desktop-tauri/.prettierrc
```

Содержимое:

```json id="mk2kqc"
{
  "semi": true,
  "singleQuote": false,
  "trailingComma": "all",
  "printWidth": 100,
  "tabWidth": 2,
  "endOfLine": "lf"
}
```

Почему так:

```text id="ih19i8"
semi: true — меньше спорных edge cases в TypeScript;
singleQuote: false — проще для JSX и JSON-like strings;
trailingComma: all — чище diff;
printWidth: 100 — удобно для React/TSX;
endOfLine: lf — стабильнее в git.
```

## 3.4. Добавить scripts в `package.json`

```json id="bijm9u"

    "typecheck": "tsc --noEmit",
```

9. Проверить, что график отображается.
10. Экспортировать Markdown report.
11. Экспортировать HTML report.
12. Сохранить JSON project.

## Команды перед коммитом

